## Verdict
A controls/stat wall dominates the first screen of BOTH pages, and it's worse than the notes page ever was. AGENDA: under the ~100px sticky header (which itself now carries a view-tab row with count pills), the main opens with a 4-tile decorative stat grid (page.tsx:461-510) that wraps to 2 cols x 2 rows on a ~390px phone — ~150px — and TWO of those four tiles (Vandaag, Wachtrij) are exact duplicates of counts already shown as badges on the view-tabs immediately above. Then comes the AgendaCalendar, whose header itself stacks vertically below lg (AgendaCalendar.tsx:138 flex-col-until-lg) for another ~110-140px before the first agenda row. Net: the timeline (the page's actual job) starts well past 700px, off-screen, and the today-count string renders THREE times within ~600px. ROOSTER is structurally worse: the entire hero `<section>` (page.tsx:442-480) — SectionHeader + full gradient NextShiftCard + a 4-tile MobileRosterSnapshot card grid — sits OUTSIDE every `tab ===` conditional, so a 'next shift' summary renders identically on Statistieken, Salaris, and Beheer where it's irrelevant, and ~350-370px of in-flow chrome buries the timeline ~480-530px down on phone. SINGLE BIGGEST CONTENT-FIRST WIN: scope rooster's hero section to the Overzicht tab only (one-line conditional move) AND collapse both pages' chunky card-tile grids into a single compact notes-style StatChip row — that one pattern swap reclaims ~150px on each page and kills the cross-tab repetition in a single stroke. The chip/disclosure pattern is already shipped and proven in NotesMetrics.tsx:75-129; this is a port, not an invention.

**Biggest win:** Swap both pages' chunky card-tile grids for the shipped notes StatChip row (a single flex-wrap row of h-8 chips with detail in tooltips) AND scope rooster's hero section to the Overzicht tab — together this reclaims ~150px on each page's first screen, kills the cross-tab and tab-vs-tile count duplication, and makes the calendar/timeline the first content under the header, all by porting a pattern that already exists in NotesMetrics.tsx rather than inventing anything new.

## Themes
- Summary-before-content inversion: both pages open with decorative stat/hero chrome and push their actual job (timeline on rooster, calendar+timeline on agenda) past the fold — the precise anti-pattern the notes page was just refactored away from.
- Chunky cards vs compact chips: rooster's MobileMetric (rounded-xl px-3 py-2.5 text-lg) and agenda's 3-line tiles (uppercase label + text-lg number + subline) are the old fat-tile language; notes' shipped StatChip (h-8 inline-flex rounded-lg, detail in title=) is the target. One shared chip component should serve all three pages.
- Count duplication: agenda shows the same counts in view-tab badges, stat tiles, AND subheaders (today count 3x within 600px); rooster shows snapshot tiles whose data restates the NextShiftCard/OverviewPanel. The chip-row consolidation that fixed notes fixes this here too.
- Scope leakage across tabs: rooster's hero renders on every tab regardless of relevance — a structural ordering bug, not just styling. Content must be conditioned on the active tab.
- Cross-page consistency gaps vs the polished notes page: safe-area-inset-top is implemented only in NotesHeader/NoteEditor; rooster and agenda headers never adopted it, so the same PWA notch fix is owed on both.
- Disclosure over always-expanded: heavy always-on bodies (full calendar month grid, full gradient hero) should collapse to compact/disclosure by default on phone, matching how notes hides detail behind tooltips and collapsed sections.

## Solid
- The notes-page StatChip pattern (NotesMetrics.tsx:75-129) is already built, professional, and directly portable — compact h-8 chips, detail in title= tooltip, onClick + aria-pressed doubling as scope filters. This is a proven template, not a from-scratch design.
- The existing stat tiles already wire onClick={() => setActiveView(...)} (agenda 463/473/484/504) and tab navigation, so the interaction wiring survives the card→chip swap untouched — only the markup shrinks.
- AgendaCalendar already defaults mobileView to 'agenda' (line 104) and keeps the tall auto-rows month grid in the hidden sm:grid desktop-only block, so phones already get the lighter list body — good responsive instinct to build on.
- Rooster correctly gates the desktop OverviewPanel behind order-2 hidden md:block (line 443), so the snapshot doesn't double-render on phone — no double-counting, the structure just needs tab-scoping and chip-ification.
- Accessibility fundamentals are in place: proper role=tablist/tab with aria-selected, aria-busy on sync, aria-label on conflict tiles, role=status for the zero-conflict case — the a11y scaffolding is solid and should be preserved through the refactor.
- NextShiftCard already ships a compact variant (NextShiftCard.tsx:100-171), so the mobile demote is a prop flip, not new component work.
